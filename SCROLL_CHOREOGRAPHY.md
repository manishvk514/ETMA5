# Scroll choreography

## Goal

The site scroll should feel authored in chapters.

## Sequence

### 1. Calibration / entry
- preloader behaves like screen calibration
- hero opens already composed
- user enters by scrolling through the framed screen

### 2. Position / conviction
- the statement is the event
- first line lands
- second line lands harder
- support copy arrives last

### 3. Work / archive
- user first sees the work as selected material
- hovering or focusing a card updates the featured stage
- this avoids a normal portfolio grid feeling

### 4. Featured project stage
- sticky region so one project can dominate
- image behaves like a physical framed plane
- metadata remains quiet and cinematic

### 5. Reel corridor
- vertical scroll translates a horizontal still corridor
- gives the sense of moving through an image archive

### 6. Method boards
- each board is a separate beat
- no simple fade-up list behavior

### 7. Conviction pause
- the site gets quieter here
- viewer sits with the statement before entering contact

### 8. Intake room
- final section feels selective and calm

## Motion rules

- no bouncy springs
- no floating decorative loops
- use masks, crops, depth shifts, and light sweeps instead
- keep response fast but reveals measured
